## 一、高优先级（建议近期迭代）

### 1. Spec-Code 对账机制
- **架构章节**：第18节 - Spec-Driven 测试生命周期
- **描述**：Spec哈希比对 + 差异报告，确保需求文档与测试代码一致性
- **涉及模块**：
  - `SpecHashCalculator` — 计算Spec文档哈希
  - `CodeHashCalculator` — 计算测试代码哈希
  - `ReconciliationReport` — 对账差异报告生成
- **预估工作量**：3-5天

### 2. Shadow DOM 穿透感知
- **架构章节**：第22节 - A11ySnapshot 感知模式
- **描述**：递归遍历Shadow DOM，确保完整感知页面结构
- **涉及模块**：
  - `PerceptionEngine._traverse_shadow_dom()` — Shadow DOM递归
  - `ShadowRootCollector` — 收集所有Shadow Root
- **预估工作量**：2-3天

## 二、中优先级（中期规划）

### 3. HAR 回放模块
- **架构章节**：第11节 - 执行增强能力
- **描述**：录制网络请求为HAR文件，支持回放Mock
- **涉及模块**：
  - `HarRecorder` — 录制网络请求到HAR
  - `HarReplayer` — 从HAR文件回放Mock响应
- **预估工作量**：3-4天

### 4. Clock 时间控制
- **架构章节**：第11节 - 执行增强能力
- **描述**：控制浏览器时间，测试时间敏感场景（倒计时、定时任务）
- **涉及模块**：
  - `ClockController` — 暂停/加速/设置系统时间
- **预估工作量**：2天

### 5. 审计日志不可篡改
- **架构章节**：第29节 - 可观测性架构
- **描述**：审计日志写入不可篡改存储（区块链/签名链）
- **涉及模块**：
  - `ImmutableAuditLog` — 带签名的审计日志
  - `AuditSigner` — 日志签名验证
- **预估工作量**：5-7天

## 三、低优先级（长期规划）

### 6. Dashboard Web界面
- **架构章节**：第2节 - 核心架构（用户交互层）
- **描述**：实时测试监控Dashboard，展示执行进度、结果、趋势
- **涉及模块**：
  - `DashboardServer` — aiohttp Web服务器
  - `WebSocketHandler` — 实时事件推送
  - `DashboardUI` — 前端界面（Vue/React）
- **预估工作量**：10-15天
- **依赖**：EventBus事件订阅

### 7. SandboxManager 容器化执行
- **架构章节**：第27节 - Sandbox与容器化
- **描述**：统一沙箱管理接口，支持Docker/K8s/进程隔离
- **涉及模块**：
  - `SandboxManager` — 统一沙箱抽象
  - `DockerSandbox` — Docker容器隔离
  - `K8sSandbox` — Kubernetes Pod隔离
  - `ProcessSandbox` — 进程级隔离
  - `ResourceLimiter` — CPU/内存/网络限制
- **预估工作量**：15-20天

### 8. OpenTelemetry 可观测性
- **架构章节**：第29节 - 可观测性架构
- **描述**：集成OTel，支持分布式链路追踪、指标采集
- **涉及模块**：
  - `OtelTracer` — 链路追踪
  - `OtelMeter` — 指标采集（LLM调用/执行耗时/缓存命中率）
  - `OtelExporter` — 导出到OTLP后端
- **预估工作量**：7-10天

### 9. DAGOrchestrator 编排引擎
- **架构章节**：第30节 - 高级编排架构
- **描述**：支持DAG依赖的测试任务编排，并行执行独立任务
- **涉及模块**：
  - `DAGBuilder` — 构建测试依赖图
  - `DAGExecutor` — 拓扑排序+并行执行
  - `DependencyResolver` — 依赖解析
- **预估工作量**：10-12天

### 10. SDK 四层API
- **架构章节**：第31节 - API架构
- **描述**：标准化SDK分层（NL API / Action API / Core API / Platform API）
- **涉及模块**：
  - `NLApi` — 自然语言接口层
  - `ActionApi` — 动作原子层
  - `CoreApi` — 核心抽象层
  - `PlatformApi` — 平台适配层
- **预估工作量**：8-10天

### 11. TokenBudget 成本治理
- **架构章节**：第32节 - 成本治理架构
- **描述**：LLM Token预算控制、成本追踪、告警
- **涉及模块**：
  - `TokenBudget` — Token预算分配
  - `CostTracker` — 成本追踪
  - `CostAlert` — 超预算告警
- **预估工作量**：5-7天

### 12. CI/CD Pipeline 模板
- **架构章节**：第33节 - CI/CD集成
- **描述**：预置CI/CD Pipeline模板（GitHub Actions/Jenkins/GitLab CI）
- **涉及模块**：
  - `PipelineTemplate` — Pipeline模板生成
  - `QualityGate` — 质量门禁（覆盖率/通过率/性能）
  - `ReportPublisher` — 报告发布到CI平台
- **预估工作量**：6-8天

### 13. 多租户 RBAC+ABAC
- **架构章节**：第34节 - 多租户架构
- **描述**：支持多租户隔离，RBAC角色权限 + ABAC属性权限
- **涉及模块**：
  - `TenantManager` — 租户管理
  - `PermissionEngine` — 权限引擎
  - `RoleManager` — 角色管理（RBAC）
  - `AttributePolicy` — 属性策略（ABAC）
- **预估工作量**：15-20天

### 14. 知识图谱推理
- **架构章节**：第35节 - 知识图谱架构
- **描述**：构建测试知识图谱，支持推理和演化
- **涉及模块**：
  - `KnowledgeGraphBuilder` — 构建知识图谱
  - `GraphReasoner` — 图推理引擎
  - `GraphEvolver` — 知识演化（自动更新）
- **预估工作量**：20-25天

### 15. ModelEvaluator 评估基准
- **架构章节**：第36节 - 评估基准架构
- **描述**：模型效果评估、AB测试框架、基准测试集
- **涉及模块**：
  - `ModelEvaluator` — 模型评估器
  - `ABTestFramework` — AB测试框架
  - `BenchmarkSuite` — 基准测试集
  - `EvaluationMetrics` — 评估指标（准确率/召回率/F1）
- **预估工作量**：10-15天
