# UIAI 自动化测试框架 — 架构设计文档

> **版本**：v2.0 | **日期**：2026-06-11

## 1. 项目定位

**UIAI** = **智能自主 Agent** + **Playwright 执行层** + **多模型支持**

核心理念：**用户描述目标 → Agent自主完成测试**

## 3. 核心流程

### 3.1 智能Agent执行循环（参考browser-use）

```
用户目标: "登录系统，验证跳转到首页"
    │
    ▼
┌─────────────────────────────────────────┐
│           TestAgent.run()               │
│                                         │
│   while not done:                       │
│       │                                 │
│       ├── [感知] 获取页面状态            │
│       │       DOM / 截图 / URL          │
│       │                                 │
│       ├── [决策] LLM分析下一步           │
│       │       thinking + action[]       │
│       │                                 │
│       ├── [执行] 执行动作列表            │
│       │       click / input / navigate  │
│       │                                 │
│       ├── [后处理] 循环检测 + 目标判断    │
│       │                                 │
│       └── 目标完成 → break              │
│                                         │
└─────────────────────────────────────────┘
    │
    ▼
录制代码 + 生成报告
```

### 3.2 AgentOutput 数据模型

```python
class AgentOutput:
    thinking: str              # 内部思考
    evaluation_previous_goal: str  # 评估上一步
    memory: str                # 跨步记忆
    next_goal: str             # 下一步目标
    action: list[ActionModel]  # 动作列表（每步可多个）
    is_done: bool              # 目标是否完成
```

## 4. 四层运行模式

| 层级 | 模式 | 占比 | 说明 |
|-----|------|------|------|
| **L1** | 确定性脚本 | 60% | 手写/录制的Python代码，直调Playwright |
| **L2** | 智能Agent + DOM | 25% | Agent感知DOM → LLM决策 → 执行循环 |
| **L3** | 智能Agent + 视觉 | 10% | Agent感知截图 → VL决策 → 坐标执行 |
| **L4** | 本地开发 | 5% | MCP Server + Claude Code CLI |

---

## 5. 关键设计决策

| 决策 | 选择 | 原因 |
|------|------|------|
| **执行方式** | 进程内直调Playwright SDK | 消除MCP RPC开销，生产环境稳定 |
| **Agent循环** | 自研TestAgent（参考browser-use） | 完全可控，多模型适配 |
| **感知模式** | DOM优先，视觉降级 | DOM可录制代码，视觉覆盖Canvas等场景 |
| **LLM提供商** | OpenAI / Qwen / Ollama 多选 | 用户自由选择，无限制 |
| **循环检测** | LoopDetector跟踪历史 | 防止死循环 |
| **代码录制** | 执行同步录制 | 执行完即可回放 |

---

## 6. Agent角色分工

| Agent | 角色 | 触发时机 |
|-------|------|---------|
| **TestAgent** | 主执行 | 用户发起测试目标 |
| **VisualAgent** | 视觉执行 | DOM不可用时降级 |
| **HealerAgent** | 自愈修复 | 执行失败时触发 |
| **ExplorerAgent** | 探索测试 | 用户发起URL探索 |
| **LoopDetector** | 循环检测 | 每步后检测 |

---

## 7. CLI命令

| 命令 | 说明 |
|------|------|
| `uiai run <goal>` | 智能Agent执行测试目标 |
| `uiai replay <script>` | 回放录制脚本 |
| `uiai explore <url>` | AI探索性测试 |
| `uiai mcp` | 启动MCP Server（本地开发） |

## 9. 与browser-use对照

| 特性 | browser-use | UIAI |
|------|-------------|------|
| Agent循环 | Perceive-Reason-Act | ✅ 相同设计 |
| 每步多动作 | max_actions_per_step | ✅ 支持 |
| 循环检测 | ActionLoopDetector | ✅ LoopDetector |
| DOM序列化 | Accessibility Tree | ✅ 相同 |
| 视觉模式 | 截图+VL | ✅ 支持 |
| 代码录制 | ❌ 不支持 | ✅ 支持 |
| 多平台 | ❌ 仅Web | ✅ Web+App预留 |
| 多模型 | 绑定LangChain | ✅ 多提供商自由选择 |
